# Final Submission Checklist

Use this list before submitting your project.

## Required Files

- [ ] **README.md** – Project overview, setup, and run instructions
- [ ] **PROJECT_REPORT.md** – Objectives, methodology, results, conclusion
- [ ] **requirements.txt** – All Python dependencies
- [ ] **app.py**, **predict.py**, **train_model.py** – Application and ML code
- [ ] **model/skin_model.h5** – Trained model (or note in README if too large; provide link or training instructions)
- [ ] **model/class_names.json** – Class labels used by the app
- [ ] **model/metrics.json** – Validation accuracy (target 70–80%); generated by `python train_model.py`
- [ ] **dataset/** – Sample or full dataset, or clear instructions in DATA_README.md
- [ ] **templates/** and **static/** – Frontend files

## Before Submitting

1. **Run the app**: `python app.py` → open http://127.0.0.1:5000 and test upload + prediction.
2. **Install from scratch**: In a new folder, `pip install -r requirements.txt` and run again to confirm.
3. **Replace placeholder author/date** in README.md and PROJECT_REPORT.md if required by your institution.
4. **Zip or upload** the project folder (exclude `venv/`, `__pycache__/` – see .gitignore).

- If dataset or model is too large: upload to Drive/OneDrive and add download link in README/DATA_README.
